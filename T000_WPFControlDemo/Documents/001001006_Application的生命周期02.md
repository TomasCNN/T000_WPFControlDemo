# 001001006_Application的生命周期02

​	Application 是 WPF 程序的 “全局总控中心”，其生命周期覆盖**程序启动→运行→退出**的完整流程，每个阶段的事件是控制程序行为、保障稳定性的核心入口。以下按 “生命周期阶段” 拆解所有重要事件，包含**触发时机、核心用途、工业场景实战案例、使用注意事项**，确保新手能理解、能落地。

## 一、生命周期总览

​	Application 的生命周期可分为三大核心阶段，各阶段目标和事件逻辑清晰：

| 阶段     | 核心目标                             | 关键特征                    |
| :------- | :----------------------------------- | :-------------------------- |
| 启动阶段 | 稳定初始化（配置 / 校验 / 窗口创建） | 一次性触发，程序 “开局准备” |
| 运行阶段 | 可靠运行（监控 / 交互 / 异常处理）   | 反复触发，程序 “核心工作期” |
| 退出阶段 | 安全收尾（资源 / 数据 / 硬件清理）   | 一次性触发，程序 “最后离场” |

## 二、启动阶段：程序初始化（核心事件：Startup、LoadCompleted）

​	启动阶段是程序从 “进程创建” 到 “主窗口显示” 的过程，事件触发顺序为 `Startup` → `LoadCompleted`（若满足触发条件）。

### 1. Startup 事件（启动阶段核心）

#### 触发时机

​	程序进程启动后，**任何窗口 / 全局资源加载前** 触发（优先级高于 `App.xaml` 中配置的 `StartupUri`）。

#### 核心用途（工业软件高频场景）

- 单实例运行校验（避免多实例抢占硬件 / 数据库资源）；
- 加载全局配置（如 PLC 设备 IP、采集频率、用户权限）；
- 处理命令行参数（如双击数据文件直接启动程序并加载文件）；
- 按权限 / 环境动态创建主窗口（如管理员 / 操作员分离界面）；
- 初始化底层硬件通信（如提前建立 PLC 连接）。

#### 工业场景实战案例（数据采集软件）

csharp:

```c#
using System;
using System.Threading;
using System.Windows;

namespace IndustrialDataCollector
{
    public partial class App : Application
    {
        private void App_Startup(object sender, StartupEventArgs e)
        {
            // 1. 单实例校验（工业软件必备：防止多实例抢占PLC资源）
            bool isNewInstance;
            // 命名规则：公司标识+程序标识（避免与其他程序冲突）
            using (Mutex mutex = new Mutex(true, "MyFactory_DataCollector_Mutex", out isNewInstance))
            {
                if (!isNewInstance)
                {
                    MessageBox.Show("数据采集程序已在运行！", "提示", 
                        MessageBoxButton.OK, MessageBoxImage.Information);
                    Shutdown(); // 退出重复实例
                    return;
                }

                // 2. 加载全局配置（工业场景：从JSON读取设备参数）
                GlobalConfig.Load("device_config.json");

                // 3. 处理命令行参数（双击.dat文件，直接加载该文件）
                string targetFilePath = e.Args.Length > 0 ? e.Args[0] : null;

                // 4. 按用户权限创建不同主窗口
                MainWindow = GlobalConfig.CurrentUser.Role == "Admin" 
                    ? new AdminMainWindow(targetFilePath)  // 管理员窗口（含配置功能）
                    : new OperatorMainWindow(targetFilePath); // 操作员窗口（仅采集功能）

                // 5. 显示主窗口，完成启动
                MainWindow.Show();
            }
        }
    }
}
```

#### 关键注意事项

- 优先级：若同时配置 `StartupUri` 和 `Startup` 事件，`Startup` 事件会覆盖 `StartupUri`（`StartupUri` 失效）；
- 异常处理：此阶段抛出未捕获异常会直接导致程序启动失败，建议加 `try-catch` 兜底；
- 互斥锁：需引入 `System.Threading` 命名空间，命名要唯一（建议用 “公司域名 + 程序名”）；
- 资源释放：`using` 语句确保 Mutex 在代码块结束后释放，避免锁泄漏。

### 2. LoadCompleted 事件

#### 触发时机

​	程序**所有启动资源**（App.xaml 全局资源、启动窗口、窗口内所有控件）加载完成后触发（手动创建窗口也会触发）。

#### 核心用途

- 统计程序启动耗时（工业软件性能监控 / 优化）；
- 自动执行设备连接测试（如 PLC / 传感器连通性检测）；
- 启动后台采集线程（异步执行，不阻塞 UI）。

#### 工业场景实战案例

csharp:

```c#
using System;
using System.Threading.Tasks;
using System.Windows;

namespace IndustrialDataCollector
{
    public partial class App : Application
    {
        // 记录启动开始时间（在Startup事件中初始化）
        private DateTime _startupStartTime;

        private void App_Startup(object sender, StartupEventArgs e)
        {
            _startupStartTime = DateTime.Now; // 记录启动开始时间
            // ... 其他Startup逻辑（单实例、加载配置等）
        }

        private void App_LoadCompleted(object sender, NavigationEventArgs e)
        {
            // 1. 统计启动耗时（工业软件性能分析）
            TimeSpan startupCost = DateTime.Now - _startupStartTime;
            LogHelper.Info($"程序启动完成，总耗时：{startupCost.TotalMilliseconds:F0} 毫秒");

            // 2. 异步测试所有设备连接（避免阻塞UI）
            Task.Run(() =>
            {
                bool isAllConnected = DeviceHelper.TestAllConnections();
                // 切换回主线程更新UI
                Dispatcher.Invoke(() =>
                {
                    ((MainWindow)MainWindow).UpdateDeviceStatus(isAllConnected);
                });
            });

            // 3. 启动后台数据采集线程
            DataCollectService.Start();
        }
    }
}
```

#### 关键注意事项

- 触发条件：即使不使用 `NavigationWindow`/`Frame`，手动创建窗口也会触发此事件；
- 异步操作：耗时操作（如设备连接测试）必须用 `Task.Run` 异步执行，避免 UI 卡顿；
- 日志记录：启动耗时是工业软件性能优化的重要指标，建议持久化到日志。

## 三、运行阶段：程序工作期（核心事件：Activated、Deactivated、DispatcherUnhandledException、StartupNextInstance、ResourcesChanged）

​	运行阶段无固定起止点（从主窗口显示到触发退出条件），是程序与用户交互的核心阶段，事件多为 “状态监控” 和 “异常兜底”。

### 1. Activated 事件（程序激活）

#### 触发时机

​	程序从 “后台态” 切换到 “前台态”（获得窗口焦点）时触发（如用户点击程序窗口、Alt+Tab 切回程序）。

#### 核心用途

- 恢复高频数据采集（如 PLC 从 1 次 / 秒→10 次 / 秒）；
- 刷新界面实时数据（如设备温度、转速）；
- 重新订阅设备消息队列（如 MQTT 消息）。

#### 工业场景实战案例

csharp:

```c#
private void App_Activated(object sender, EventArgs e)
{
    // 1. 恢复PLC高频数据采集
    DataCollectService.ResumeHighFreqCollect();

    // 2. 刷新界面实时数据
    ((MainWindow)MainWindow).RefreshRealTimeData();

    // 3. 重新订阅MQTT设备消息
    MqttClientHelper.ReSubscribe();

    LogHelper.Info("程序切换到前台，恢复正常采集频率");
}
```

#### 关键注意事项

- 首次触发：程序启动后首次显示窗口时，会自动触发一次 `Activated`；
- 多窗口：多窗口程序中，切换窗口焦点也会触发；
- 性能：避免在此事件中执行耗时操作，防止用户切换窗口时卡顿。

### 2. Deactivated 事件（程序失活）

#### 触发时机

​	程序从 “前台态” 切换到 “后台态”（失去窗口焦点）时触发（如用户点击其他软件、Alt+Tab 切走）。

#### 核心用途

- 降低数据采集频率（节省 CPU / 网络资源）；
- 暂停非必要 UI 动画（如数据趋势图动效）；
- 临时保存采集缓存（防止异常丢失）。

#### 工业场景实战案例

csharp:

```c#
private void App_Deactivated(object sender, EventArgs e)
{
    // 1. 降低PLC采集频率（从10次/秒→1次/秒）
    DataCollectService.PauseHighFreqCollect();

    // 2. 暂停界面动画
    ((MainWindow)MainWindow).PauseAnimation();

    // 3. 临时保存采集缓存到本地
    DataCacheHelper.SaveTempCache();

    LogHelper.Info("程序切换到后台，降低采集频率");
}
```

#### 关键注意事项

- 仅失焦：此事件触发不代表程序退出，核心采集逻辑建议 “降频不停止”，避免数据断档；
- 缓存保存：临时缓存建议保存到内存 / 本地临时文件，避免频繁写硬盘。

### 3. DispatcherUnhandledException 事件（全局异常兜底）

#### 触发时机

​	程序**主线程**抛出未被局部 `try-catch` 捕获的异常时触发（子线程异常不触发）。

#### 核心用途

- 全局异常兜底（防止程序闪退）；
- 记录详细异常日志（含设备 ID、时间等上下文）；
- 显示友好提示（屏蔽技术细节，符合工业软件用户体验）；
- 紧急保存采集数据（避免数据丢失）。

#### 工业场景实战案例

csharp:

```c#
private void App_DispatcherUnhandledException(object sender, System.Windows.Threading.DispatcherUnhandledExceptionEventArgs e)
{
    // 1. 记录完整异常日志（含堆栈、设备信息）
    LogHelper.Error(
        $"全局未处理异常：{e.Exception.Message}", 
        e.Exception, 
        new { DeviceId = GlobalConfig.CurrentDevice.Id, Time = DateTime.Now });

    // 2. 显示友好提示（工业场景：不暴露技术细节）
    MessageBox.Show(
        "数据采集异常，请检查设备连接或联系管理员！", 
        "运行错误", 
        MessageBoxButton.OK, 
        MessageBoxImage.Error);

    // 3. 紧急保存未提交的采集数据
    DataCollectService.SaveEmergencyData();

    // 4. 标记异常已处理，防止程序崩溃
    e.Handled = true;
}
```

#### 关键注意事项

- 仅主线程：子线程异常需手动捕获，或通过 `Dispatcher.Invoke` 抛到主线程；
- 核心参数：`e.Handled = true` 是防止程序崩溃的关键（不设置则程序仍会闪退）；
- 日志完整性：工业场景需记录设备 ID、采集参数等上下文，方便排查问题。

### 4. StartupNextInstance 事件（单实例程序专属）

#### 触发时机

​	仅 “单实例程序” 生效：用户尝试启动第二个程序实例时，**第一个实例**触发此事件（新实例自动退出）。

#### 核心用途

- 激活已有主窗口（替代 “弹窗提示退出”，更友好）；
- 传递新实例的命令行参数（如双击数据文件让已有程序打开）。

#### 工业场景实战案例

csharp:

```
// 前提：Visual Studio中配置单实例→项目属性→应用→勾选“单实例应用”
private void App_StartupNextInstance(object sender, StartupNextInstanceEventArgs e)
{
    // 1. 激活已有主窗口（最小化则恢复）
    if (MainWindow.WindowState == WindowState.Minimized)
    {
        MainWindow.WindowState = WindowState.Normal;
    }
    MainWindow.Activate(); // 调到前台

    // 2. 获取新实例的命令行参数（如双击的.dat文件路径）
    if (e.CommandLine.Count > 0)
    {
        string filePath = e.CommandLine[0];
        // 让已有程序加载该文件
        ((MainWindow)MainWindow).LoadDataFile(filePath);
    }

    // 3. 标记将窗口带到前台
    e.BringToForeground = true;
}
```

#### 关键注意事项

- 配置前提：必须先将程序设为 “单实例应用”，否则此事件不触发；
- 新实例行为：新实例会自动退出，无需手动调用 `Shutdown()`；
- 参数校验：处理命令行参数时需校验路径合法性（如文件是否存在）。

### 5. ResourcesChanged 事件（全局资源变更）

#### 触发时机

​	`Application.Current.Resources`（全局资源池）发生变更时触发（新增 / 删除 / 修改资源）。

#### 核心用途

- 动态换肤后刷新界面样式；
- 多语言切换后更新全局文案；
- 监控资源变更，避免样式失效。

#### 工业场景实战案例

csharp:

```c#
public partial class App : Application
{
    private void App_Startup(object sender, StartupEventArgs e)
    {
        // 手动订阅ResourcesChanged事件（App.xaml中无法直接绑定）
        Application.Current.ResourcesChanged += App_ResourcesChanged;
        // ... 其他Startup逻辑
    }

    private void App_ResourcesChanged(object sender, ResourcesChangedEventArgs e)
    {
        // 1. 记录资源变更日志
        LogHelper.Info($"全局资源变更：{e.ChangeType} - {e.ResourceKey}");

        // 2. 动态换肤：刷新主窗口样式
        if (e.ResourceKey.ToString() == "MainThemeBrush")
        {
            ((MainWindow)MainWindow).ApplyNewTheme();
        }

        // 3. 多语言切换：更新全局文案
        if (e.ResourceKey.ToString().StartsWith("Lang_"))
        {
            ((MainWindow)MainWindow).UpdateLanguageText();
        }
    }
}
```

#### 关键注意事项

- 订阅方式：需手动在代码中订阅（App.xaml 无法直接绑定）；
- 作用域：仅监控全局资源，窗口 / 控件级资源变更不触发；
- 适用场景：工业软件 “免重启换肤 / 换语言” 的核心事件。

## 四、退出阶段：程序收尾（核心事件：SessionEnding、Exit）

​	退出阶段是程序从 “触发退出条件” 到 “进程终止” 的过程，事件触发顺序为 `SessionEnding`（系统级退出时）→ `Exit`。

### 1. SessionEnding 事件（系统级退出预警）

#### 触发时机

​	Windows 系统会话结束前触发（用户注销、关机、重启、远程会话断开），**早于 Exit 事件**。

#### 核心用途

- 紧急保存未提交的采集数据；
- 断开硬件连接（PLC / 传感器，避免设备异常）；
- 提示用户确认收尾操作。

#### 工业场景实战案例

csharp:

```
private void App_SessionEnding(object sender, SessionEndingCancelEventArgs e)
{
    // 1. 判断是否有未保存数据
    if (DataCollectService.HasUnsavedData)
    {
        // 2. 弹出紧急提示
        MessageBoxResult result = MessageBox.Show(
            "系统即将关机/注销，是否紧急保存采集数据？", 
            "紧急提示", 
            MessageBoxButton.YesNo, 
            MessageBoxImage.Warning);

        if (result == MessageBoxResult.Yes)
        {
            // 3. 同步保存数据（确保完成，避免异步未执行）
            DataCollectService.SaveEmergencyData();
            // 4. 断开所有设备连接
            DeviceHelper.DisconnectAll();
        }
    }

    // 可选：临时取消会话结束（仅延迟，系统最终仍会关机）
    // e.Cancel = true;
}
```

#### 关键注意事项

- 执行顺序：此事件触发后，无论是否取消，最终都会执行 `Exit` 事件；
- 保存方式：紧急数据需**同步保存**（避免异步操作未完成导致数据丢失）；
- 硬件断开：工业场景必须断开硬件连接，防止设备处于 “持续通信” 异常状态。

### 2. Exit 事件（程序最终收尾）

#### 触发时机

​	程序触发退出条件后（主窗口关闭、调用 `Shutdown()`、系统关机），**所有窗口关闭前** 触发。

#### 核心用途

- 保存用户配置（窗口大小、采集参数、主题偏好）；
- 关闭数据库 / 硬件连接；
- 记录退出日志（审计 / 故障排查）。

#### 工业场景实战案例

csharp:

```c#
private void App_Exit(object sender, ExitEventArgs e)
{
    // 1. 保存用户配置（如窗口大小、采集频率）
    GlobalConfig.Save("user_config.json");

    // 2. 关闭数据库连接
    DbHelper.CloseConnection();

    // 3. 断开剩余硬件连接（兜底）
    DeviceHelper.DisconnectAll();

    // 4. 记录退出日志（含退出码）
    LogHelper.Info($"程序正常退出，退出码：{e.ApplicationExitCode}");
}
```

#### 关键注意事项

- 退出码：`e.ApplicationExitCode` 可自定义（0 = 正常退出，非 0 = 异常退出），方便日志排查；
- 兜底处理：即使 `SessionEnding` 中已断开连接，`Exit` 仍需兜底，避免遗漏；
- 资源释放：所有持久化连接（数据库、MQTT）需在此处显式关闭。

## 五、核心补充：事件执行顺序与关键规则

### 1. 整体执行顺序（完整流程）

```html
程序启动 → Startup → 加载全局资源/窗口 → LoadCompleted → 运行阶段（Activated/Deactivated反复触发）→ 
（触发退出）→ SessionEnding（系统级退出时）→ Exit → 程序进程终止
```

### 2. 关键规则

- ShutdownMode：默认 `OnMainWindowClose`（主窗口关闭则退出），可配置为 `OnExplicitShutdown`（仅调用 `Shutdown()` 退出，适合多窗口场景）；
- 异常传播：子线程异常不会触发 `DispatcherUnhandledException`，需手动捕获；
- 资源释放：所有系统级资源（Mutex、数据库连接）需在 `Exit` 或 `using` 中释放，避免泄漏。

## 总结

### 核心事件记忆要点

1. **启动必用**：`Startup`（初始化核心）、`LoadCompleted`（启动后收尾）；
2. **运行必监控**：`DispatcherUnhandledException`（全局异常兜底）、`Activated/Deactivated`（资源动态调整）；
3. **退出必处理**：`SessionEnding`（系统级紧急收尾）、`Exit`（通用资源清理）；
4. **工业加分项**：`StartupNextInstance`（单实例友好交互）、`ResourcesChanged`（动态换肤 / 多语言）。

​	这些事件覆盖了 Application 生命周期的全维度，合理使用能让工业软件实现 “稳定启动、可靠运行、安全退出” 的核心目标，也是 WPF 程序工程化开发的基础。
