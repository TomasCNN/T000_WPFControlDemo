# 001001_App.xaml与Application类

xaml类型的文件包含两部分，一部分以.xaml扩展名结尾的前端代码，另一部分以.xaml.cs结尾的后端代码，通常我们也把后端代码称为隐藏代码。

前端代码的全称为Extensible Application Markup Language，简称XAML。

## WPF 中 App.xaml 与 Application 类（通俗易懂解读）

​	在 WPF 程序里，`App.xaml` 和 `Application` 类是**整个应用程序的 “总控中心”** —— 简单说：

- `App.xaml` 是「应用级的配置文件」（XAML 形式），负责定义应用的全局资源、启动窗口、事件等；
- `Application` 类是「应用程序的代码后台」（C# 类），对应`App.xaml`，负责管理应用的生命周期（启动、退出、异常）、全局状态和资源。

你可以把它们理解成：

👉 `App.xaml` = 应用的「全局配置清单」（写在 XAML 里，可视化）；

👉 `Application` 类 = 应用的「总控逻辑代码」（执行清单里的配置，处理全局事件）。

### 一、核心关系：App.xaml ↔ Application 类

#### 1. 自动关联（无需手动配置）

​	新建 WPF 项目时，VS 会自动生成：

- `App.xaml`（XAML 文件）：根节点是`<Application>`，指定对应的后台类；
- `App.xaml.cs`（代码文件）：定义`App`类，继承自`System.Windows.Application`，是`App.xaml`的后台逻辑。

##### 示例：自动生成的关联代码

xaml：

```xaml
<!-- App.xaml：x:Class指定后台类 -->
<Application x:Class="WpfApp1.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             StartupUri="MainWindow.xaml"> <!-- 指定启动窗口 -->
    <Application.Resources>
        <!-- 全局资源（所有窗口都能用） -->
    </Application.Resources>
</Application>
```

csharp：

```c#
// App.xaml.cs：后台类，继承Application
namespace WpfApp1
{
    /// <summary>
    /// App.xaml 的交互逻辑
    /// </summary>
    public partial class App : Application
    {
        // 可在这里写应用级的逻辑
    }
}
```

#### 2. 核心作用（为什么需要它们）

| 组件           | 核心作用                                                     | 通俗例子                                                     |
| :------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| App.xaml       | 1. 指定应用启动的第一个窗口；2. 定义全局资源（样式、模板、字符串等）；3. 声明应用级事件（启动、退出、异常） | 就像 “软件的全局设置文件”，规定启动时打开哪个窗口，所有窗口共用的样式 |
| Application 类 | 1. 管理应用生命周期（启动、运行、退出）；2. 访问 / 修改全局资源；3. 处理应用级异常、命令行参数；4. 管理所有打开的窗口 | 就像 “软件的总控程序”，负责启动、监控运行状态、处理全局错误  |

### 二、App.xaml 的核心用法

#### 1. 指定启动窗口（最基础）

​	通过`StartupUri`属性，指定应用启动时自动打开的窗口（无需写代码）：

xaml:

```xaml
<!-- App.xaml：启动MainWindow.xaml -->
<Application ... StartupUri="MainWindow.xaml">
</Application>
```

👉 效果：运行程序时，自动创建并显示`MainWindow`窗口。

#### 2. 定义全局资源（所有窗口共用）

​	在`<Application.Resources>`中定义的资源（如样式、颜色、模板），可被整个应用的所有窗口使用，无需重复定义。

##### 示例：全局样式（所有按钮共用）

xaml:

```xaml
<Application ...>
    <Application.Resources>
        <!-- 全局按钮样式：所有窗口的按钮都用这个样式 -->
        <Style TargetType="Button">
            <Setter Property="Width" Value="100"/>
            <Setter Property="Height" Value="30"/>
            <Setter Property="Background" Value="#007ACC"/>
            <Setter Property="Foreground" Value="White"/>
        </Style>

        <!-- 全局颜色常量：可绑定使用 -->
        <SolidColorBrush x:Key="MainColor" Color="#FF6B8E23"/>

        <!-- 全局字符串：多语言/统一文案 -->
        <system:String x:Key="AppTitle">工业数据采集系统</system:String>
    </Application.Resources>
</Application>
```

##### 窗口中使用全局资源（直接引用）

xaml：

```
<!-- MainWindow.xaml：使用App.xaml的全局资源 -->
<Window ...>
    <Grid>
        <!-- 按钮自动应用全局样式 -->
        <Button Content="测试按钮"/>

        <!-- 引用全局颜色 -->
        <Ellipse Fill="{StaticResource MainColor}" Width="50" Height="50"/>

        <!-- 引用全局字符串 -->
        <TextBlock Text="{StaticResource AppTitle}" FontSize="20"/>
    </Grid>
</Window>
```

#### 3. 声明应用级事件（XAML 方式）

​	在`App.xaml`中声明应用的生命周期事件，绑定到后台方法：

xaml

```xaml
<Application ... 
             Startup="App_Startup"          <!-- 应用启动时触发 -->
             Exit="App_Exit"                <!-- 应用退出时触发 -->
             DispatcherUnhandledException="App_DispatcherUnhandledException"> <!-- 未捕获异常时触发 -->
</Application>
```

### 三、Application 类的核心用法（App.xaml.cs）

#### 1. 处理应用生命周期事件（核心）

​	在`App.xaml.cs`中实现`App.xaml`声明的事件，控制应用行为：

csharp:

```c#
using System;
using System.Windows;

namespace WpfApp1
{
    public partial class App : Application
    {
        // 1. 应用启动时触发（替代StartupUri，更灵活）
        private void App_Startup(object sender, StartupEventArgs e)
        {
            // 可在这里处理命令行参数、初始化配置、动态选择启动窗口
            Console.WriteLine("应用开始启动...");

            // 替代StartupUri：手动创建并显示窗口（更灵活）
            MainWindow mainWindow = new MainWindow();
            mainWindow.Show();

            // 处理命令行参数（比如通过参数指定启动窗口）
            if (e.Args.Length > 0 && e.Args[0] == "admin")
            {
                AdminWindow adminWindow = new AdminWindow();
                adminWindow.Show();
            }
        }

        // 2. 应用退出时触发（清理资源）
        private void App_Exit(object sender, ExitEventArgs e)
        {
            Console.WriteLine("应用即将退出...");
            // 清理资源：关闭数据库连接、保存配置、释放内存
            ConfigHelper.SaveConfig();
            DbHelper.CloseConnection();
        }

        // 3. 捕获应用中未处理的异常（防止程序崩溃）
        private void App_DispatcherUnhandledException(object sender, System.Windows.Threading.DispatcherUnhandledExceptionEventArgs e)
        {
            // 显示错误提示
            MessageBox.Show($"程序发生错误：{e.Exception.Message}", "错误", MessageBoxButton.OK, MessageBoxImage.Error);
            // 标记异常已处理，防止程序崩溃
            e.Handled = true;
        }
    }
}
```

#### 2. 访问全局资源（代码方式）

​	通过`Application.Current.Resources`访问`App.xaml`中定义的全局资源：

csharp:

```c#
// 在任意窗口/控件中访问全局资源
private void Button_Click(object sender, RoutedEventArgs e)
{
    // 获取全局颜色
    SolidColorBrush mainColor = (SolidColorBrush)Application.Current.Resources["MainColor"];
    // 获取全局字符串
    string appTitle = (string)Application.Current.Resources["AppTitle"];

    MessageBox.Show($"应用标题：{appTitle}");
}
```

#### 3. 管理应用窗口（全局控制）

​	`Application.Current`是`Application`的全局单例，可在任意位置访问，管理所有窗口：

csharp:

```c#
// 1. 获取当前激活的窗口
Window activeWindow = Application.Current.MainWindow;

// 2. 遍历所有打开的窗口（比如关闭所有窗口退出应用）
private void CloseAllWindows()
{
    // 反向遍历，避免删除时索引异常
    for (int i = Application.Current.Windows.Count - 1; i >= 0; i--)
    {
        Application.Current.Windows[i].Close();
    }
}

// 3. 退出应用（触发Exit事件）
private void ExitApp()
{
    Application.Current.Shutdown(); // 正常退出
    // Application.Current.Shutdown(1); // 带退出码退出（1表示异常退出）
}
```

#### 4. 自定义全局属性 / 方法（应用级共享）

在`App`类中定义静态属性 / 方法，实现全局数据共享（无需传参）：

csharp:

```c#
public partial class App : Application
{
    // 全局用户信息（所有窗口都能访问）
    public static User CurrentUser { get; set; }

    // 全局方法（比如显示全局提示）
    public static void ShowGlobalTip(string msg)
    {
        MessageBox.Show(msg, "提示", MessageBoxButton.OK, MessageBoxImage.Information);
    }
}

// 在任意窗口中调用
private void LoginSuccess(User user)
{
    // 设置全局用户
    App.CurrentUser = user;
    // 调用全局方法
    App.ShowGlobalTip($"欢迎{user.Name}登录！");
}
```

### 四、工业场景常用技巧

#### 1. 禁用多实例运行（只允许打开一个程序）

​	在`App_Startup`中实现，避免用户多次启动程序：

csharp:

```c#
private void App_Startup(object sender, StartupEventArgs e)
{
    // 检查是否已有实例运行
    bool isNewInstance;
    using (System.Threading.Mutex mutex = new System.Threading.Mutex(true, "WpfApp1_Mutex", out isNewInstance))
    {
        if (isNewInstance)
        {
            // 新实例：正常启动
            MainWindow mainWindow = new MainWindow();
            mainWindow.Show();
        }
        else
        {
            // 已有实例：提示并退出
            MessageBox.Show("程序已在运行中！", "提示", MessageBoxButton.OK, MessageBoxImage.Information);
            Shutdown(); // 退出当前实例
        }
    }
}
```

#### 2. 全局异常日志（工业软件必备）

​	在`App_DispatcherUnhandledException`中记录异常日志，方便排查问题：

csharp:

```c#
private void App_DispatcherUnhandledException(object sender, System.Windows.Threading.DispatcherUnhandledExceptionEventArgs e)
{
    // 记录异常到日志文件（用log4net/NLog等库）
    LogHelper.Error("未处理异常", e.Exception);
    // 显示友好提示
    MessageBox.Show("程序发生异常，请查看日志！", "错误", MessageBoxButton.OK, MessageBoxImage.Error);
    e.Handled = true;
}
```

#### 3. 动态加载全局资源（按需切换）

工业软件中可根据配置动态加载不同的全局样式（比如白天 / 黑夜模式）：

csharp:

```c#
// 切换到黑夜模式
private void SwitchToDarkMode()
{
    // 清除原有资源
    Application.Current.Resources.MergedDictionaries.Clear();
    // 加载黑夜模式资源字典
    ResourceDictionary darkDict = new ResourceDictionary();
    darkDict.Source = new Uri("Resources/DarkStyle.xaml", UriKind.Relative);
    Application.Current.Resources.MergedDictionaries.Add(darkDict);
}
```

### 五、核心区别与总结

#### 1. App.xaml vs Application 类

| 维度     | App.xaml                            | Application 类                           |
| :------- | :---------------------------------- | :--------------------------------------- |
| 形式     | XAML 声明式配置                     | C# 命令式代码                            |
| 核心     | 定义 “是什么”（启动窗口、全局资源） | 实现 “怎么做”（启动逻辑、异常处理）      |
| 适用场景 | 静态配置（固定资源、默认启动窗口）  | 动态逻辑（参数处理、条件启动、异常捕获） |

#### 2. 关键点回顾

1. **核心定位**：`App.xaml`是应用的「全局配置文件」，`Application`类是「全局逻辑处理类」，两者一一对应；
2. **生命周期**：通过`Startup`/`Exit`事件控制应用启动 / 退出，`DispatcherUnhandledException`捕获全局异常；
3. **全局资源**：`App.xaml`中定义的资源可被所有窗口使用，代码中通过`Application.Current.Resources`访问；
4. **全局控制**：`Application.Current`是单例，可在任意位置管理窗口、退出应用、访问全局数据；
5. **工业场景**：重点用它实现「单实例运行」「全局异常日志」「动态资源切换」「命令行参数处理」。

​	简单记：`App.xaml`管 “静态配置”，`Application`类管 “动态逻辑”，两者结合是 WPF 应用的 “总控中心”，负责全局的配置、生命周期和资源管理。
